Bad Example Page

This page Explains a bad example how it will affect the security of the page.

Form

A form is one of the best examples where things can go wrong as thats the only medium where usually a website user inputs a certain pience of information which can be personal or confidential.

The above example is one of the simple and one out of many examples where things can go wrong in terms of login. When the user inputs the username and password, if you notice the url it will display the username and the password to the url. This is a serious issue and it is due to the use of GET instead of POST method on the Form. These information can be stored on the browser history which can create a security issue.

Another issue is with the validation. The current setup contains only basic front end validation backend validation for the the input data. This will result in a malicious user to take advantage and inject an sql injection attack attack by just deleting the front end validation using the dev tools. Hence GET request should ne avoided in case of forms where there is password involved.
